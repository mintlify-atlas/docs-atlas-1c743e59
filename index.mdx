## What is Cardinal?

Cardinal is a powerful command-line tool and library for analyzing procurement data using the [Open Contracting Data Standard (OCDS)](https://standard.open-contracting.org/). It calculates procurement indicators and detects red flags that may indicate corruption, collusion, or other irregularities in public procurement processes.

Built in Rust for high performance, Cardinal can process large datasets efficiently and provides both a CLI interface and library bindings for Rust and Python.

## Quick Example

Analyze procurement data in three simple steps:

<Steps>
  <Step title="Initialize settings">
    Create a configuration file with default settings for all indicators:
    
    ```bash
    ocdscardinal init settings.ini
    ```
  </Step>
  
  <Step title="Prepare your data">
    Correct quality issues and standardize your OCDS data:
    
    ```bash
    ocdscardinal prepare data.jsonl --settings settings.ini --output prepared.jsonl --errors errors.csv
    ```
  </Step>
  
  <Step title="Calculate indicators">
    Run red flag analysis on your prepared data:
    
    ```bash
    ocdscardinal indicators --settings settings.ini prepared.jsonl > results.json
    ```
  </Step>
</Steps>

## Use Cases

Cardinal is designed for:

- **Anti-corruption agencies** investigating procurement fraud
- **Civil society organizations** monitoring public spending
- **Government oversight bodies** auditing contracting processes
- **Researchers** analyzing procurement patterns and behaviors
- **Procurement professionals** improving process quality
